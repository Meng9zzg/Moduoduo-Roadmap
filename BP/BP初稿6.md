
## Slide 11: Runtime 的价值

Moduoduo Runtime 解决的不是“功能不够多”，而是三种失控：

| 失控类型 | 传统方案 | Runtime 的解法 |
|------|------|------|
| **交互失控** | 多人、打断、弱网、回流直接崩 | 场景状态机 + Mode Engine |
| **模型失控** | 幻觉、时效错误、来源混乱 | 多模型路由 + RAG + MTE 时间语义引擎 |
| **运维失控** | 现场升级难、回滚难、监控难 | Ops Plane + 弱网降级 + 崩溃自愈 |

> **Runtime 的核心价值，不是让 AI 更花哨，而是让 AI 终于能上岗。**

## Slide 14: Global Gateway

AI 世界正在分裂为三个阵营：

- 美国模型
- 中国模型
- 开源 / 私有模型

客户不会永远只使用一个阵营。
真实业务往往同时面对五个问题：

- 性能
- 合规
- 成本
- 地域
- 结算

Gateway 不能只是“接模型的模块”，而必须成为一个**跨阵营调度层**。

一个真正的跨阵营调度层，必须部署在同时满足以下条件的中立节点：

- 中国模型能稳定到达
- 国际模型能稳定到达
- 企业客户能合规结算
- 海外项目能持续服务

> **新加坡不是出海包装，而是 Gateway 的工程解。**
>
> **Global Gateway 是一项核心产品能力，而不是附属模块。**

对客户而言，它解决的是一件更本质的事：

- 当业务跨区域展开时，模型选择不再受单一阵营限制
- 当合规、性能、成本发生冲突时，可以进行统一调度
- 当节点增长时，Gateway 会成为整个 Runtime Network 的入口层

Runtime 负责在现实空间稳定运行 AI。
Global Gateway 负责把全球 AI 能力稳定接入这套系统。
